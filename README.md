# Snakes

**Snakes** — это учебный проект на C++: расширенная версия классической игры «Змейка» с несколькими игроками, ботами, кроликами, жизнями, сердечками здоровья, ASCII-режимом, графическим режимом на SFML и турнирным режимом до трёх побед.

Проект хорошо подходит для демонстрации работы с ООП, разделением логики игры и отображения, обработкой событий, простым искусственным интеллектом, CMake-сборкой и парсингом аргументов командной строки через CLI11.

## Описание программы

Игра представляет собой поле, на котором одновременно могут находиться несколько змей. Часть змей может управляться игроками, а часть — ботами. На поле появляются кролики, которых змеи могут съедать для увеличения длины. Также с небольшой вероятностью появляются сердечки здоровья. Если змея подбирает сердечко, количество её жизней увеличивается на одну.

По умолчанию у каждой змейки **3 жизни**. Если змея врезается в стену, себя или другую змею, она теряет одну жизнь. Если жизни ещё остались, змея возрождается с начальной длиной. Если жизни закончились, она удаляется из игры.

Игра поддерживает два режима отображения:

1. **Графический режим** на SFML.
2. **Терминальный ASCII-режим**.

Также есть **турнирный режим**, где раунды идут до тех пор, пока одна из змей не наберёт 3 победы. Между раундами отображается таблица счёта.

## Сборка и запуск

### Зависимости

Для сборки проекта нужны:

- C++20-compatible compiler, например `g++` или `clang++`.
- CMake версии 3.16 или выше.
- SFML 2.6.
- CLI11.

На Linux зависимости можно установить примерно так:

```bash
sudo apt update
sudo apt install cmake g++ libsfml-dev libcli11-dev
```

Название пакета CLI11 может отличаться в зависимости от дистрибутива. Если пакет не находится, CLI11 можно подключить вручную или установить через пакетный менеджер вашей системы.

### Сборка

Из корня проекта:

```bash
mkdir -p build
cd build
cmake ..
cmake --build .
```

После сборки исполняемый файл будет находиться в папке `build`.

### Запуск

```bash
./Snakes
```

## Параметры командной строки

| Параметр | Тип | Значение по умолчанию | Описание |
|---|---:|---:|---|
| `--graphics` | `bool` | `true` | Включает графический режим. Если указать `false`, игра запустится в ASCII-режиме в терминале. |
| `--tournament` | `bool` | `false` | Включает турнирный режим. В турнире раунды идут до 3 побед одной змейки. |
| `--width` | `uint32_t` | `60` | Ширина игрового поля в клетках. Минимальное значение автоматически ограничивается до `20`. |
| `--height` | `uint32_t` | `30` | Высота игрового поля в клетках. Минимальное значение автоматически ограничивается до `10`. |
| `--num_humans` | `size_t` | `2` | Количество змей, управляемых игроками. Сейчас управление предусмотрено для двух игроков. |
| `--num_bots` | `size_t` | `3` | Количество змей-ботов. |
| `--num_rabbits` | `size_t` | `5` | Максимальное количество кроликов на поле. |

## Мини-статьи про основные фичи

### 1. Обычный игровой режим

В обычном режиме создаётся игровая модель, выбранное количество змей, кроликов и нужный вид отображения. После этого запускается игровой цикл. На каждом шаге игра:

1. Обрабатывает события клавиатуры.
2. Перерисовывает состояние поля.
3. Обновляет модель, если игра не на паузе.
4. Передвигает змей.
5. Проверяет поедание кроликов и сердечек.
6. Проверяет столкновения.
7. Создаёт новых кроликов и сердечки, если это возможно.

Такой подход отделяет игровую логику от отображения. Модель отвечает за правила игры, а view-классы отвечают только за вывод на экран.

---

### 2. Графический режим на SFML

Графический режим включён по умолчанию:

```bash
./Snakes --graphics=true
```

В этом режиме используется окно SFML. Каждая клетка игрового поля переводится в пиксели. Змеи отображаются цветными прямоугольниками, кролики — маленькими фигурками, а сердечки здоровья — красными сердцами.

В верхней части окна находится preview-панель. На ней показываются цвета змей и количество их жизней в виде сердечек. Это удобно, потому что игрок сразу видит, сколько попыток осталось у каждой змейки.

---

### 3. ASCII-режим в терминале

ASCII-режим можно включить так:

```bash
./Snakes --graphics=false
```

В этом режиме игра рисуется прямо в терминале с помощью ANSI escape-последовательностей. Поле, змеи, кролики и сердечки отображаются текстовыми символами.

ASCII-режим полезен тем, что не требует графического окна. Он показывает, что логика игры не зависит от конкретного способа отображения. Одна и та же модель может быть отрисована и через SFML, и через терминал.

---

### 4. Кролики

Кролики — это основная еда для змей. На поле поддерживается заданное максимальное количество кроликов. Если змейка съедает кролика, её тело увеличивается на один сегмент.

Количество кроликов задаётся параметром:

```bash
./Snakes --num_rabbits=10
```

Чем больше кроликов, тем быстрее змеи могут расти. В режиме с ботами это делает игру более динамичной, потому что боты постоянно ищут ближайшую цель.

---

### 5. Сердечки здоровья

Сердечки здоровья — отдельная сущность игры. Они появляются на свободных клетках поля с небольшой вероятностью. По умолчанию на поле может быть только одно сердечко одновременно.

Если змейка подбирает сердечко, количество её жизней увеличивается на одну. Сердечки появляются реже, чем кролики, поэтому они являются ценным бонусом, а не обычной едой.

Эта механика делает игру менее случайной и более интересной: даже после ошибки игрок может продолжить раунд, если заранее накопил дополнительные жизни.

---

### 6. Система жизней

По умолчанию у каждой змейки 3 жизни. Если змея сталкивается со стеной, своим телом или другой змеёй, она теряет одну жизнь.

Если после потери жизни у змейки ещё есть жизни, она возрождается. При возрождении её тело возвращается к стандартной длине, а направление становится начальным. Если стандартная точка появления занята, игра пытается найти другую свободную область для возрождения.

Если жизни закончились, змейка удаляется из игры.

---

### 7. Боты

Боты управляются автоматически. Каждый бот выбирает направление движения на основе простой оценки возможных ходов.

Алгоритм учитывает:

- нельзя идти в противоположную сторону;
- нельзя выходить за границы поля;
- нельзя идти в занятую клетку;
- желательно двигаться ближе к ближайшему кролику;
- желательно выбирать клетки, вокруг которых больше свободного пространства.

Такой бот не является идеальным, но он достаточно хорошо подходит для турниров и демонстрации базового искусственного интеллекта в игре.

---

### 8. Турнирный режим

Турнирный режим запускается так:

```bash
./Snakes --tournament=true
```

В турнире игра состоит из отдельных раундов. Раунд заканчивается, когда на поле остаётся одна змейка. Эта змейка получает `+1` к счёту.

Турнир продолжается до тех пор, пока одна из змей не наберёт 3 победы. Перед каждым раундом и после завершения турнира показывается таблица результатов.

Пример запуска турнира с двумя игроками и тремя ботами:

```bash
./Snakes --graphics=true --tournament=true --num_humans=2 --num_bots=3
```

---

### 9. Таблица счёта

В турнирном режиме между раундами появляется scoreboard-экран. В нём отображаются:

- цвет змейки;
- название змейки;
- текущее место в таблице;
- количество побед.

Список сортируется по количеству побед. Если у нескольких змей одинаковый счёт, они дополнительно сортируются по названию.

---

### 10. Настройка размера поля

Размер поля задаётся в клетках, а не в пикселях:

```bash
./Snakes --width=80 --height=40
```

Это удобно, потому что логика игры работает именно с клетками. В графическом режиме каждая клетка просто масштабируется в пиксели.

Минимальная ширина поля — 20 клеток. Минимальная высота — 10 клеток. Если передать значения меньше, программа автоматически увеличит их до минимально допустимых.

## Архитектура

### `GameModel`

`GameModel` хранит состояние игры:

- размеры поля;
- список змей;
- список кроликов;
- список сердечек здоровья;
- максимальное количество кроликов;
- вероятность появления сердечек;
- указатели на змей, управляемых людьми.

Также `GameModel` отвечает за обновление игрового состояния: движение змей, обработку еды, столкновения, возрождение и удаление змей.

### `Snake`

Класс `Snake` описывает одну змейку. Он хранит:

- тело змеи;
- направление движения;
- цвет;
- точку появления;
- тип управления;
- количество жизней.

Змейка умеет двигаться, менять направление, терять жизнь, получать жизнь и возрождаться.

### `Rabbit`

`Rabbit` — простая сущность еды. Она хранит только позицию на поле. Когда змея съедает кролика, тело змеи увеличивается.

### `Health`

`Health` — бонусная сущность. Она тоже хранит позицию на поле. Когда змея подбирает сердечко, количество её жизней увеличивается.

### `Game`

`Game` управляет игровым циклом. Он связывает модель и отображение:

- получает события от view;
- передаёт управление нужным змейкам;
- вызывает `render`;
- вызывает `update` у модели;
- следит за окончанием раунда в турнирном режиме.

### `IView`

`IView` — общий интерфейс для отображения. Благодаря этому игра может работать с разными видами интерфейса, не меняя игровую логику.

Сейчас реализованы два view-класса:

- `AsciiView`;
- `GraphicsView`.

## Краткий итог

Этот проект — не просто классическая змейка, а расширенная версия игры с несколькими режимами, ботами, турниром, жизнями и бонусами.
